# Задание №4 — Динамическая загрузка и выполнение методов через рефлексию 
Необходимо расширить приложение файловой системы возможностью динамической загрузки внешних библиотек (плагинов) и выполнения их методов через графический интерфейс. \
Программа использует механизмы рефлексии .NET для анализа структуры классов, создания их экземпляров и вызова методов с параметрами, введенными пользователем во время выполнения 

**Механизм рефлексии (ReflectionViewModel):** \
-Загружает сборку (.dll) по указанному пути пользователя. \
-Сканирует типы сборки и фильтрует их по реализации IFileSystemPlugin. \
-Динамически формирует список доступных классов и их публичных методов. \
-Автоматически генерирует поля ввода для каждого параметра выбранного метода. \
-Преобразует строковые значения из UI в требуемые типы данных (int, long, string[] и т.д.). \
-Создает экземпляр класса через Activator.CreateInstance и вызывает метод через MethodInfo.Invoke.

**Что используется в проекте:** \
Язык программирования — C# \
Платформа — .NET 8.0 \
UI фреймворк — Avalonia UI \
MVVM библиотека — CommunityToolkit.Mvvm \

**Загрузка библиотеки плагинов** \
-Перейдите на вкладку «Рефлексия». \
-В поле «Путь к DLL» укажите полный путь к скомпилированной библиотеке (например: ...\FileSystem.Core\bin\Debug\net8.0\FileSystem.Core.dll). \
-Нажмите кнопку «Загрузить». \
-В списке «Классы» появятся все классы, реализующие IFileSystemPlugin. 

**Выбор класса и метода** \
-Выберите нужный класс из списка (например, FileSystemAnalyzer). \
-В правом списке «Методы» отобразятся все доступные методы выбранного класса. \
-Кликните на название метода (например, BuildPath).

**Ввод параметров и выполнение** \
-В нижней панели «📝 Параметры метода» автоматически появятся поля ввода для каждого аргумента. \
Примечание: Для массивов строк (params string[]) вводите значения через запятую (например: Docs, Photos, Work). \
-Заполните поля соответствующими значениями. \
-Нажмите зеленую кнопку «▶ Выполнить метод». \
-В нижнем текстовом блоке отобразится результат выполнения метода или сообщение об ошибке, если типы данных не совпали.
